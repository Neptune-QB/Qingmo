## 难点3：ASR转录错误沿链路逐级放大

高光点和剧情摘要的质量取决于 LLM 收到的输入文本。一旦 ASR 把人名、关键台词转错，整条流水线就建立在错误地基上——但 LLM 本身不会质疑输入。

### 坑一：同音人名无人纠正，下游全链路污染

FunASR 对中文同音字几乎无分辨能力。以《北派寻宝笔记》ep1063 为例：

| 正确文本 | FunASR 输出 |
|----------|------------|
| 项云峰 | 向云峰 |
| 项爷 | 相爷 / 向爷 |
| 我入行本就是个错误 | 我入套木盆就是错误 |

问题不在 FunASR——问题在于后续链路。LLM 拿到 "向云峰""相爷" 这种输入后，不会质疑、不会纠正，直接当作真实角色名写入窗口摘要、高光点描述、每集摘要：

```
ASR: "向云峰被围堵"
  → episode_scene_segment.summary: "向云峰遭到众人围堵威胁"
  → drama_highlight.title: "向云峰当众驳斥讨债方"
  → episode_content_summary.short_summary: "向云峰出狱遭遇逼迫..."
  → 向量 RAG 检索: 用户搜"项云峰" → 语义匹配得分降低
```

影响范围：全部 5 张 RAG 相关表、827 条窗口摘要、53 条高光点、23 条每集摘要——共 **170 处**同音错字污染。

### 坑二：幻觉台词逐级放大，LLM 照单全收

ASR 将 "我入行本就是个错误" 误识别为 "我入套木盆就是错误" 后，LLM 不质疑这串无意义文本，照常在窗口摘要中分析："主角承认自己陷入了木盆圈套"。更隐蔽的是，LLM 甚至会围绕这个幻觉编造一整套叙事——"木盆指的可能是江湖行话，隐喻某种入局仪式"——从一句错字衍生出一段不存在的情节。

### 解决方案（四层防线）

**① LLM 后纠错环节 (DeepSeek, 150 句 / 15s)**

ASR 输出 → DeepSeek 逐句纠错 prompt（含专为短剧设计的修正指令）→ 重点修正同音错字、断句错误、罕见词幻觉 → 纠错后回写 transcripts。

实测效果：ep1063 的 150 句 ASR 台词经 DeepSeek 纠错后，核心人名和关键台词准确率显著提升，仅"锒铛入狱"等成语仍偶有漏修。

**② 角色名同音字映射表暴力预替换**

在 LLM 纠错之前，先用硬编码映射表扫一遍全部台词文本：

```
_HOMOPHONE_FIX_MAP = {
    "向云峰": "项云峰", "相云峰": "项云峰", "向爷": "项爷", "相爷": "项爷", ...
}
```

`_pre_fix_character_names()` 在 LLM 分析前执行，确保高频角色名已全部修正到位。

**③ 剧情背景注入纠错 prompt**

纠错 prompt 中强制注入剧名 + 简介 + 已知角色名单，给 DeepSeek 提供"正确答案"参照系：

> 本剧《北派寻宝笔记》是一部盗墓志怪题材短剧。已知主要角色：项云峰（男主，盗墓世家传人）、于枫、李静、李大全。请据此修正以下 ASR 转录文本中的同音错字和识别错误。

**④ 纠错后回写，后续分析基于干净文本**

纠错结果直接 `UPDATE episode_transcript`，窗口分析、高光判定、每集摘要全部基于干净 text 进行——不是透传原始 ASR，而是用修正后的文本。

### 小结

ASR 错误传播的本质是流水线的"信任链条"问题——每个下游环节无条件信任上游输入。解决思路不是让每个环节都质疑输入，而是在离源头最近的位置（ASR → 纠错）一次性拦截，保证后续链路的数据地基稳固。
